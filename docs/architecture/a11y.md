# Accessibility (WCAG 2.2 AA)

StudyForge is free for everyone — that includes screen-reader users,
keyboard-only users, and people on low-contrast displays. WCAG 2.2 AA
is the floor, not the ceiling.

## How we audit today (Phase 4)

The lightweight runner ships with the web app. Enable it by setting
`NEXT_PUBLIC_AXE_DEV=1` in `apps/web/.env.local` and reloading any page
under `/(app)`. A green ⌫ pill in the bottom-right shows zero
violations; red shows the count. Click it to dump the full list to the
browser console.

The runner is `apps/web/components/axe-reporter.tsx`. It does NOT
ship in production builds (gated on `NODE_ENV !== 'production'`).

## Manual checklist for new components

- **Inputs**: every `<input>` / `<textarea>` / `<select>` must have an
  `htmlFor` label OR `aria-label`.
- **Icon-only buttons**: must carry `aria-label` (e.g. the search trigger,
  the budget pill).
- **Modals**: `role="dialog"`, `aria-modal="true"`, an `aria-label` or
  `aria-labelledby`. Escape closes; focus traps to the modal.
- **Color contrast**: text-foreground on bg-background hits 4.5:1; the
  muted-foreground secondary text hits 3:1. Tier pills (emerald / amber /
  rose) use semantic backgrounds — never rely on colour alone to convey
  state (every pill has a text label).
- **Keyboard nav**: Tab through the whole flow; nothing skipped, no
  invisible focus rings on focusable elements.

## CI-gated Playwright + axe (Phase 5)

Phase 5 hardening adds `@axe-core/playwright` against a fixed route set
— `/`, `/dashboard`, `/courses/[id]`, `/settings/byok`, `/instructor` —
with a violations-must-equal-zero gate. The dev runner here is the
human-in-the-loop precursor.
